# 🛡️ Валидация и конфигурация

## Валидация полей ввода

Система автоматически проверяет корректность вводимых данных.

### Типы валидации

#### 1. **Текстовые поля (названия)**

**Правила:**
- ✅ Должны содержать буквы (русские или латинские)
- ✅ Могут содержать цифры
- ❌ Не могут состоять только из цифр
- ❌ Не могут быть пустыми

**Примеры:**

| Ввод | Результат | Сообщение |
|------|-----------|-----------|
| `Борщ` | ✅ OK | |
| `Борщ №5` | ✅ OK | |
| `Картофель фри` | ✅ OK | |
| `123` | ❌ ОШИБКА | "Название не может состоять только из цифр" |
| ` ` (пусто) | ❌ ОШИБКА | "Поле не может быть пустым" |
| `#$%` | ❌ ОШИБКА | "Название должно содержать буквы" |

**Где применяется:**
- Название поставщика
- Название ингредиента
- Название блюда
- Название группы блюд
- Название подразделения
- ФИО менеджера

---

#### 2. **Числовые поля (цены, количество)**

**Правила:**
- ✅ Только цифры и десятичная точка/запятая
- ✅ Должны быть > 0 (если не указано иное)
- ❌ Не могут быть отрицательными
- ❌ Не могут содержать буквы или спецсимволы

**Примеры:**

| Поле | Ввод | Результат | Сообщение |
|------|------|-----------|-----------|
| Цена | `150.50` | ✅ OK | |
| Цена | `0` | ❌ ОШИБКА | "Значение должно быть больше нуля" |
| Цена | `-50` | ❌ ОШИБКА | "Значение не может быть отрицательным" |
| Цена | `abc` | ❌ ОШИБКА | "Поле должно содержать только числа" |
| Цена | `100!` | ❌ ОШИБКА | "Поле должно содержать только числа" |
| Остаток | `0` | ✅ OK | (ноль допустим для остатков) |
| Остаток | `-10` | ❌ ОШИБКА | "Значение не может быть отрицательным" |

**Где применяется:**
- Цена блюда
- Цена закупки
- Количество товара
- Объём продаж
- Остаток на складе (может быть 0)

---

#### 3. **Проценты (наценка)**

**Правила:**
- ✅ Число от 0 до 100
- ❌ Не могут быть отрицательными
- ❌ Не могут быть > 100

**Примеры:**

| Ввод | Результат | Сообщение |
|------|-----------|-----------|
| `20` | ✅ OK | |
| `99.5` | ✅ OK | |
| `0` | ✅ OK | |
| `150` | ❌ ОШИБКА | "Процент должен быть от 0 до 100" |
| `-5` | ❌ ОШИБКА | "Процент должен быть от 0 до 100" |

**Где применяется:**
- Наценка на товар

---

#### 4. **Поле "Выход" (для блюд)**

**Правила:**
- ✅ Должно содержать единицу измерения (буквы)
- ✅ Может содержать числа
- ❌ Не может быть пустым
- ❌ Не может содержать отрицательные числа

**Примеры:**

| Ввод | Результат | Сообщение |
|------|-----------|-----------|
| `1 порция` | ✅ OK | |
| `250 г` | ✅ OK | |
| `300 мл` | ✅ OK | |
| `порция` | ✅ OK | |
| `1` | ❌ ОШИБКА | "Выход должен содержать единицу измерения" |
| `-1 порция` | ❌ ОШИБКА | "Значение не может быть отрицательным" |
| ` ` (пусто) | ❌ ОШИБКА | "Поле не может быть пустым" |

**Где применяется:**
- Выход блюда

---

#### 5. **ИНН**

**Правила:**
- ✅ Только цифры
- ✅ Длина 10 или 12 символов
- ✅ Может быть пустым

**Примеры:**

| Ввод | Результат | Сообщение |
|------|-----------|-----------|
| `7743013902` | ✅ OK | (10 цифр) |
| `773401390283` | ✅ OK | (12 цифр) |
| ` ` (пусто) | ✅ OK | |
| `123` | ❌ ОШИБКА | "ИНН должен содержать 10 или 12 цифр" |
| `abc1234567890` | ❌ ОШИБКА | "ИНН должен содержать только цифры" |

**Где применяется:**
- ИНН поставщика

---

#### 6. **Телефон**

**Правила:**
- ✅ Цифры, пробелы, +, -, (, )
- ✅ Может быть пустым
- ❌ Не может содержать буквы или другие символы

**Примеры:**

| Ввод | Результат | Сообщение |
|------|-----------|-----------|
| `+7 (495) 123-45-67` | ✅ OK | |
| `89161234567` | ✅ OK | |
| `123-456` | ✅ OK | |
| ` ` (пусто) | ✅ OK | |
| `8(916)abc-def` | ❌ ОШИБКА | "Телефон может содержать только цифры, +, -, ( )" |

**Где применяется:**
- Телефон менеджера поставщика

## Визуальная индикация ошибок

При ошибке валидации:

1. **Красная рамка** вокруг поля
2. **Текст ошибки** справа от поля (красным цветом)
3. **Tooltip** при наведении на поле с полным текстом ошибки
4. **MessageBox при сохранении** — если есть ошибки, появится информационное окно со списком всех проблем

### При попытке сохранить с ошибками

Если вы нажмёте кнопку "💾 Сохранить" при наличии ошибок валидации, появится информационное окно:

```
┌────────────────────────────────────────────────┐
│  ℹ  Ошибка валидации                          │
├────────────────────────────────────────────────┤
│                                                │
│  Обнаружены ошибки в заполнении полей:        │
│                                                │
│  • Название не может состоять только из цифр  │
│  • Значение должно быть больше нуля           │
│  • ИНН должен содержать 10 или 12 цифр        │
│                                                │
│                            ┌────────┐          │
│                            │   OK   │          │
│                            └────────┘          │
└────────────────────────────────────────────────┘
```

**Данные не будут сохранены** пока вы не исправите все ошибки.

## ⚙️ Конфигурационный файл

### Файл `app.conf`

Позволяет настроить название окна и иконку **без перекомпиляции программы**.

**Расположение:** рядом с `CateringIS.exe`

### Формат файла

```ini
# Комментарии начинаются с #

# Название главного окна
WindowTitle=Название вашего заведения

# Путь к иконке (относительно папки с программой)
WindowIcon=app.ico
```

### Примеры настройки

#### Для кафе:
```ini
WindowTitle=Кафе "Солнышко" - Система учёта
WindowIcon=cafe_icon.ico
```

#### Для столовой:
```ini
WindowTitle=Столовая №5 - Учётная система
WindowIcon=canteen_icon.ico
```

#### Для ресторана:
```ini
WindowTitle=Ресторан "Вкусная еда" - Управление
WindowIcon=restaurant_icon.ico
```

### Как изменить иконку

1. Положите свой файл `.ico` в папку с программой
2. Отредактируйте `app.conf`:
   ```ini
   WindowIcon=my_custom_icon.ico
   ```
3. Перезапустите программу

**Важно:** 
- Файл иконки должен быть в формате `.ico`
- Путь указывается относительно папки с программой
- Если файл не найден, используется иконка по умолчанию

### Автоматическое создание конфига

Если файл `app.conf` не существует, он создастся автоматически при первом запуске со значениями по умолчанию.

## 🔧 Техническая информация

### Реализация валидации

Валидация реализована через:
- **ValidationRule** классы в `Helpers/ValidationRules.cs`
- **Binding** с `ValidationRules` в XAML
- **ErrorTemplate** для визуализации ошибок

### Классы валидации

| Класс | Назначение |
|-------|------------|
| `NameValidationRule` | Проверка текстовых полей (названий) |
| `PositiveNumberValidationRule` | Проверка положительных чисел |
| `PercentValidationRule` | Проверка процентов (0-100) |
| `PhoneValidationRule` | Проверка телефонных номеров |

### Пример использования в XAML

```xml
<TextBox>
    <TextBox.Text>
        <Binding Path="Selected.Name" UpdateSourceTrigger="PropertyChanged">
            <Binding.ValidationRules>
                <helpers:NameValidationRule/>
            </Binding.ValidationRules>
        </Binding>
    </TextBox.Text>
</TextBox>
```

### Реализация конфига

**Файл:** `Helpers/AppConfig.cs`

```csharp
public class AppConfig
{
    public static AppConfig Instance { get; }
    public string WindowTitle { get; }
    public string WindowIcon { get; }
}
```

**Применение в MainWindow.xaml.cs:**

```csharp
var config = AppConfig.Instance;
Title = config.WindowTitle;
Icon = LoadIcon(config.WindowIcon);
```

## 💡 Советы

### Умная система ID

Программа использует **умную систему генерации ID** с переиспользованием:

**Как это работает:**
1. При создании новой записи ей присваивается первый свободный ID
2. При удалении записи её ID становится доступным для переиспользования
3. Нет пропусков в нумерации

**Пример:**
```
Создали группы: 1, 2, 3, 4, 5, 6, 7
Удалили группу 3
Создали новую → получит ID = 3 (переиспользование)
Создали ещё одну → получит ID = 8 (следующий свободный)
```

**Преимущества:**
- ✅ Нет пропусков в ID
- ✅ Компактная нумерация
- ✅ Предсказуемое поведение

### Валидация

1. **При ошибке валидации** кнопка "Сохранить" покажет информационное окно со списком ошибок

2. **Tooltip с ошибкой** появляется при наведении на поле с красной рамкой

3. **Конфиг читается один раз** при запуске - после изменения `app.conf` нужно перезапустить программу

4. **Резервная копия конфига** - рекомендуется сохранить копию настроенного `app.conf`, чтобы не потерять настройки
