## 🔧 Технические требования

### C#
- ✅ Язык: **C# 12** (.NET 8)
- ✅ Синтаксис: современный C# (pattern matching, records-ready, etc.)

### ООП
- ✅ **Классы**: `Supplier`, `Ingredient`, `Product`, `Order`, `Delivery`, etc.
- ✅ **Наследование**: `ObservableObject` (базовый класс для всех моделей)
- ✅ **Инкапсуляция**: приватные поля + публичные свойства
- ✅ **Полиморфизм**: переопределение методов, интерфейсы

### MVVM (Model-View-ViewModel)
- ✅ **Model**: папка `Models/` — чистые POCO-объекты
- ✅ **View**: папка `Views/` — XAML без code-behind логики
- ✅ **ViewModel**: папка `ViewModels/` — логика представления, команды
- ✅ **Binding**: двусторонняя привязка данных
- ✅ **Commands**: `RelayCommand` для действий пользователя
- ✅ **INotifyPropertyChanged**: для реактивности UI

### WPF
- ✅ **XAML-разметка** для всех окон
- ✅ **DataBinding** для привязки данных
- ✅ **DataTemplates** для автоматической маршрутизации View
- ✅ **Styles** и **Resources** в App.xaml
- ✅ **ValueConverters** для преобразований
- ✅ **Navigation** через ContentPresenter

## 🚀 Проверка работы

### Демонстрация основного функционала:

1. **Запустить приложение**
   ```bash
   cd CateringIS
   dotnet run
   ```

2. **Добавить поставщика**
   - Перейти в "Поставщики"
   - Нажать "+ Добавить"
   - Заполнить все поля
   - Нажать "Сохранить"

3. **Добавить ингредиенты**
   - Перейти в "Ингредиенты"
   - Добавить несколько продуктов
   - Указать поставщика

4. **Создать блюдо**
   - Перейти в "Меню"
   - Добавить блюдо
   - Загрузить фото (проверить копирование в Photos/)
   - Добавить рецептуру
   - Сохранить

5. **Приход товара**
   - Перейти в "Приход товара"
   - Создать поставку
   - Проверить обновление остатков в "Ингредиенты"

6. **Заказ подразделения**
   - Перейти в "Заказы"
   - Создать заказ
   - Проверить списание со склада

7. **Закрыть и открыть приложение**
   - Убедиться что все данные сохранились
   - Проверить папку `Data/` рядом с exe на наличие JSON-файлов

## ✨ Особенности реализации

1. **Автоматические остатки**
   - Приход → прибавляет к StockBalance
   - Заказ → вычитает из StockBalance
   - Всё через единый механизм

2. **Реактивный UI**
   - Изменения в ViewModel → автоматически в View
   - Через INotifyPropertyChanged
   - Без ручного обновления

3. **Независимость слоёв**
   - Model не знает о View
   - View не знает о Model
   - Связь через ViewModel и Binding

4. **Расширяемость**
   - Легко добавить новую сущность
   - Следовать паттерну: Model → ViewModel → View → DataTemplate
