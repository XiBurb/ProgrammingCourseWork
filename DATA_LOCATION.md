# 📁 Где находятся мои данные?

Все данные приложения сохраняются в папке **`Data/`** рядом с программой.

## Расположение при разработке

После компиляции (`dotnet run` или F5 в Visual Studio):

```
CateringIS/
└── bin/
    └── Debug/
        └── net8.0-windows/
            ├── CateringIS.exe
            └── Data/                    ← ЗДЕСЬ ВСЕ ДАННЫЕ
                ├── suppliers.json
                ├── ingredients.json
                ├── products.json
                ├── deliveries.json
                ├── orders.json
                ├── sales.json
                ├── departments.json
                ├── groups.json
                ├── meta.json
                └── Photos/
                    ├── product_1_20250220_143052.jpg
                    └── ...
```

## Расположение после публикации

После команды `dotnet publish`:

```
<куда_опубликовали>/
├── CateringIS.exe
└── Data/                    ← ЗДЕСЬ ВСЕ ДАННЫЕ
    ├── suppliers.json
    ├── ingredients.json
    └── Photos/
```

## Преимущества такого подхода

✅ **Легко найти** — всё в одной папке с программой  
✅ **Легко скопировать** — просто копируете папку `Data/`  
✅ **Резервное копирование** — архивируете `Data/` и всё сохранено  
✅ **Переносимость** — скопировали программу вместе с `Data/` на другой компьютер — всё работает  
✅ **Нет путаницы** — не нужно искать в скрытых папках Windows

## Что делать при переносе на другой компьютер?

1. Скопировать всю папку с программой
2. Папка `Data/` скопируется вместе с ней
3. Все ваши данные будут на месте!

## Структура JSON файлов

Все файлы сохраняются в читаемом формате с отступами (Formatting.Indented).

**Пример** `products.json`:
```json
[
  {
    "Id": 1,
    "Name": "Борщ украинский",
    "GroupId": 2,
    "Cost": 150.0,
    "Yield": "1 порция (250 г)",
    "PreparationDescription": "Варить свеклу...",
    "PhotoPath": "Data/Photos/product_1_20250220_143052.jpg",
    "Recipe": [
      {
        "IngredientId": 5,
        "IngredientName": "Свекла",
        "GrossWeight": 100.0,
        "NetWeight": 85.0
      }
    ]
  }
]
```

Можно открыть любой JSON файл в текстовом редакторе и посмотреть/отредактировать данные вручную.

---

**Совет:** Периодически делайте копию папки `Data/` для резервного копирования!
