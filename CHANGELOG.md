## Версия 1.3 — Улучшенная валидация и умные ID

**Дата:** 21 февраля 2026

### ✨ Новые возможности

#### 1. **MessageBox при ошибках валидации**

Теперь при попытке сохранить данные с ошибками появляется информационное окно:

```
┌────────────────────────────────────────────────┐
│  ℹ  Ошибка валидации                          │
├────────────────────────────────────────────────┤
│  Обнаружены ошибки в заполнении полей:        │
│                                                │
│  • Название не может состоять только из цифр  │
│  • Значение должно быть больше нуля           │
│                                                │
│                            ┌────────┐          │
│                            │   OK   │          │
│                            └────────┘          │
└────────────────────────────────────────────────┘
```

**Данные не сохраняются** пока не исправлены все ошибки.

#### 2. **Умная система ID с переиспользованием**

**Было:** ID росли бесконечно, даже после удаления (1, 2, 3... удалили 2... новый получает 4)

**Стало:** ID переиспользуются и нет пропусков

**Пример:**
```
Есть записи: 1, 2, 3, 4, 5, 6, 7
Удалили запись с ID = 3
Создаём новую → получает ID = 3 ✅
Создаём ещё → получает ID = 8
```

**Преимущества:**
- Нет пропусков в нумерации
- Компактные ID
- Предсказуемое поведение

#### 3. **Расширенная валидация полей**

**Новые проверки:**

**Поставщики:**
- ✅ Название — не только цифры
- ✅ Адрес — не только цифры
- ✅ ФИО менеджера — не только цифры
- ✅ Банк — не только цифры
- ✅ ИНН — 10 или 12 цифр, только цифры

**Блюда (Меню):**
- ✅ Выход — должен содержать единицу измерения (порция, г, мл)
- ✅ Не может быть отрицательным
- ✅ Не может быть пустым

**Группы блюд:**
- ✅ Название — не только цифры

**Подразделения:**
- ✅ Название — не только цифры

## Версия 1.2 — Валидация и конфигурация

**Дата:** 20 февраля 2026

### ✨ Новые возможности

#### 1. **Валидация полей ввода**

**Автоматическая проверка корректности данных:**

- **Текстовые поля** (названия):
  - ✅ Должны содержать буквы
  - ❌ Не могут состоять только из цифр
  - ❌ Не могут быть пустыми
  
- **Числовые поля** (цены, количество):
  - ✅ Только положительные числа
  - ❌ Не могут быть отрицательными или нулевыми
  - ❌ Не могут содержать буквы/спецсимволы
  
- **Проценты** (наценка):
  - ✅ От 0 до 100
  
- **Телефон**:
  - ✅ Цифры и символы +, -, (, )
  - ❌ Без букв

**Визуализация ошибок:**
- Красная рамка вокруг поля
- Текст ошибки справа от поля
- Tooltip с описанием при наведении

**Где применяется:**
- Поставщики: название, телефон
- Ингредиенты: название, наценка, остаток
- Блюда: название, цена
- Все другие текстовые и числовые поля

#### 2. **Конфигурационный файл `app.conf`**

**Настройка приложения без перекомпиляции:**

```ini
# Название главного окна
WindowTitle=Кафе "Солнышко" - Система учёта

# Путь к иконке
WindowIcon=my_icon.ico
```

**Возможности:**
- Изменение названия окна
- Настройка пути к иконке
- Автоматическое создание при первом запуске
- Комментарии в файле для удобства

**Применение:**
- Отредактируйте `app.conf`
- Перезапустите программу
- Готово!

## Версия 1.1 — Локальное хранение данных

## ✅ Что изменилось

### Новое расположение данных

**Было:** `C:\Users\Имя\AppData\Roaming\CateringIS\` (скрытая системная папка)  
**Стало:** `<папка_с_программой>/Data/` (рядом с exe файлом)

### Преимущества нового подхода

✅ **Легко найти данные** — просто откройте папку с программой  
✅ **Легко сделать резервную копию** — скопируйте папку `Data/`  
✅ **Переносимость** — скопируйте программу вместе с `Data/` на флешку или другой ПК  
✅ **Прозрачность** — все файлы на виду, можно открыть и посмотреть  
✅ **Нет путаницы** — не нужно искать в скрытых папках AppData

### Структура папки с программой

```
CateringIS/
├── CateringIS.exe           ← Исполняемый файл
├── app.ico                  ← Иконка
├── ... (другие dll и файлы)
└── Data/                    ← ВСЕ ВАШИ ДАННЫЕ ЗДЕСЬ
    ├── suppliers.json       ← Поставщики
    ├── ingredients.json     ← Склад
    ├── products.json        ← Блюда с рецептами
    ├── deliveries.json      ← Приходы
    ├── orders.json          ← Заказы
    ├── sales.json           ← Продажи
    ├── departments.json     ← Подразделения
    ├── groups.json          ← Группы блюд
    ├── meta.json            ← Служебные данные
    └── Photos/              ← Фотографии блюд
        ├── product_1_20250220_143052.jpg
        ├── product_2_20250220_144523.png
        └── ...
```

### Что нужно сделать при обновлении?

**Ничего!** При первом запуске папка `Data/` создастся автоматически.

Если у вас уже есть данные в старом AppData, можете:
1. Найти папку `C:\Users\Имя\AppData\Roaming\CateringIS\`
2. Скопировать всё содержимое в новую папку `Data/` рядом с программой
3. Готово!
